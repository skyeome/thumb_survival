# PRD: 유튜브 썸네일 어그로 감별사 테스트

## 1. 프로젝트 개요

### 1.1 배경 및 기회

유튜브 시청자들은 매일 수백 개의 썸네일에 노출되며, 무의식적으로 "어떤 썸네일이 더 클릭을 유도하는가"에 대한 직관을 축적하고 있다. 이 암묵적 지식을 게임화하면, **지적 허영심**과 **경쟁 본능**이라는 두 가지 강력한 심리적 동인을 동시에 자극할 수 있다.

바이럴 테스트 콘텐츠(MBTI 테스트, 이상형 월드컵 등)는 이미 검증된 트래픽 파이프라인이며, 본 프로젝트는 여기에 **즉시 피드백 루프**와 **도발적 결과 칭호**를 결합하여 공유 전환율을 극대화한다.

### 1.2 제품 한 줄 정의

> 실제 유튜브 썸네일 두 개를 비교하여 조회수가 더 높은 영상을 맞히는 20라운드 이진 선택 게임으로, 최종 성적 기반의 B급 칭호를 부여하여 소셜 공유를 유도하는 웹 서비스.

### 1.3 핵심 목표

| 목표             | 측정 지표(KPI)                         | 목표치                    |
| ---------------- | -------------------------------------- | ------------------------- |
| 바이럴 확산      | 결과 공유율 (Share Rate)               | 플레이 완료 유저 중 30%+  |
| 유저 리텐션      | 재방문율 (D1 Retention)                | 15%+                      |
| 광고 수익 극대화 | 세션당 광고 노출 (Impressions/Session) | 6~8회 (정책 준수 범위 내) |
| 완주율           | 20라운드 완료율                        | 70%+                      |

## 2. 타겟 유저

### 2.1 주요 페르소나

**페르소나 A — "유튜브 헤비 유저" (18~34세)**

- 하루 평균 유튜브 시청 2시간 이상
- 알고리즘 추천에 민감하고, 트렌드 콘텐츠를 빠르게 소비
- "나는 유튜브를 잘 안다"는 자부심 보유
- 결과를 인스타 스토리, 카카오톡으로 즉시 공유하는 행동 패턴

**페르소나 B — "밈 소비자" (15~28세)**

- B급 유머와 도발적 콘텐츠에 높은 반응도
- 테스트/퀴즈형 콘텐츠를 주기적으로 소비 및 공유
- "유튜브에 인생을 낭비한 침팬지" 같은 칭호에 웃으며 자발적 확산

### 2.2 사용 환경

| 환경        | 비중(예상) | 비고                           |
| ----------- | ---------- | ------------------------------ |
| 모바일 웹   | 75%        | 카카오톡/인스타 유입 메인 채널 |
| 데스크톱 웹 | 20%        | 직접 URL 접속 및 검색 유입     |
| 태블릿      | 5%         | 반응형 대응                    |

## 3. 핵심 기능 요구사항

### 3.1 게임 플로우

```
[랜딩 페이지] → [게임 시작] → [라운드 1~20 반복] → [결과 화면] → [공유/재도전]
                                │
                          매 라운드마다:
                     썸네일 A vs B 표시
                     → 유저 선택
                     → 정답/오답 즉시 피드백
                     → 실제 조회수 공개
                     → [다음 라운드] 버튼 클릭
                     → 다음 라운드 페이지 로드 (새 광고 자연 노출)
```

### 3.2 기능 상세

#### F1. 랜딩 페이지

| 항목          | 상세                                                 |
| ------------- | ---------------------------------------------------- |
| **목적**      | 흥미 유발 및 즉시 게임 시작 유도                     |
| **필수 요소** | 서비스 타이틀, 한 줄 설명, "테스트 시작" CTA 버튼    |
| **선택 요소** | 누적 플레이 수 카운터(사회적 증거), 평균 정답률 표시 |
| **UX 원칙**   | 클릭 1회로 게임 진입 (회원가입/로그인 불필요)        |
| **광고 슬롯** | 하단 앵커 광고 1개 (AdSense Auto ads 또는 수동 배치) |

| 항목                    | 상세                                                 |

| **라운드 전환**         | 유저가 "다음 라운드" 버튼을 명시적으로 클릭하여 전환 |

| 항목                 | 상세                                                                  |

| **다음 라운드 전환** | 피드백 확인 후 유저가 "다음" 버튼 클릭 시 전환 (자동 전환 없음)       |

| 항목                 | 상세                                |
| -------------------- | ----------------------------------- |
| **프로그레스 바**    | 상단 고정, 20단계 중 현재 위치 표시 |
| **현재 스코어**      | 정답 수 / 전체 라운드 실시간 표시   |
| **연속 정답 스트릭** | 3연속 정답 이상 시 🔥 이펙트 표시   |

#### F5. 결과 화면

| 항목          | 상세                                                        |
| ------------- | ----------------------------------------------------------- |
| **점수 표시** | X / 20 (정답률 %)                                           |
| **칭호 부여** | 점수 구간별 도발적 B급 칭호 (섹션 3.3 참조)                 |
| **결과 카드** | OG 이미지용 결과 카드 자동 생성 (칭호 + 점수 + 서비스 로고) |
| **공유 버튼** | 카카오톡, 인스타 스토리, X(트위터), 링크 복사               |
| **부가 CTA**  | "다시 도전하기", "친구에게 도전장 보내기"                   |

### 3.3 칭호 체계

| 점수 구간 | 칭호                             | 톤                             |
| --------- | -------------------------------- | ------------------------------ |
| 20/20     | 🏆 상위 0.1% 알고리즘의 지배자   | 경외감 + 자랑 유도             |
| 18~19     | 🧠 유튜브 마스터 알고리즘        | 우월감 자극                    |
| 15~17     | 👀 클릭의 예언자                 | 긍정적 인정                    |
| 12~14     | 🎯 평범한 유튜브 시민            | 자조적 유머                    |
| 9~11      | 🐒 유튜브에 인생을 낭비한 침팬지 | B급 자극 (핵심 공유 구간)      |
| 6~8       | 🪨 알고리즘이 포기한 자          | 도발적 유머                    |
| 0~5       | 💀 디지털 원시인                 | 극단적 자극 (역설적 공유 유도) |

> **설계 의도**: 중하위권(6~11점) 칭호를 가장 자극적으로 설계. 이 구간의 유저가 "억울함"이나 "웃김"으로 가장 높은 공유율을 보일 것으로 기대.

## 4. 콘텐츠 설계

### 4.1 썸네일 데이터 소싱

| 소싱 방식            | 설명                                           | 우선순위  |
| -------------------- | ---------------------------------------------- | --------- |
| **YouTube Data API** | 실제 유튜브 영상의 썸네일 + 조회수 크롤링      | P0 (MVP)  |
| **수동 큐레이션**    | 에디터가 직접 비교가 재미있는 썸네일 쌍을 선별 | P0 (MVP)  |
| **AI 생성 썸네일**   | 가상의 자극적 썸네일을 AI로 생성하여 출제      | P2 (확장) |

### 4.2 문제 구성 원칙

- **조회수 차이 밸런스**: 너무 뻔하지도, 너무 랜덤하지도 않은 조회수 차이 (10%~300% 범위)
- **장르 다양성**: 먹방, 브이로그, 게임, 뉴스, 교육 등 장르 믹스
- **난이도 커브**: 초반 5문제는 비교적 쉽게 → 후반으로 갈수록 난이도 상승
- **문제 풀(Pool)**: 최소 100쌍 이상 확보하여 20문제를 랜덤 추출 (재방문 시 중복 최소화)
- **로테이션**: 주 1회 이상 신규 문제 추가, 시의성 있는 트렌드 영상 반영

## 5. 수익화 전략 (애드센스 정책 완전 준수)

### 5.1 애드센스 정책 핵심 준수 사항

본 프로젝트의 광고 설계는 Google AdSense 프로그램 정책을 전면 준수한다. 아래는 본 서비스에 직접 해당되는 정책 요약과 그에 대한 대응이다.

| 정책 항목                       | AdSense 규정                                                                  | 본 서비스 대응                                                                          |
| ------------------------------- | ----------------------------------------------------------------------------- | --------------------------------------------------------------------------------------- |
| **자동 새로고침 금지**          | 퍼블리셔는 유저의 요청 없이 페이지나 페이지 요소를 새로고침할 수 없음         | 모든 라운드 전환은 유저의 명시적 클릭("다음" 버튼)으로만 발생. 자동 전환 없음           |
| **우발적 클릭 유도 금지**       | 메뉴, 네비게이션, 다운로드 링크로 오인될 수 있는 위치에 광고를 배치할 수 없음 | 광고와 클릭 가능 요소(썸네일) 사이 최소 150px 간격 확보. 광고는 콘텐츠 영역 외부에 배치 |
| **콘텐츠 대비 광고 비율**       | 광고가 콘텐츠보다 많아서는 안 됨                                              | 페이지당 광고 슬롯 최대 2개. 콘텐츠(썸네일+제목+UI)가 화면의 70% 이상 차지              |
| **부자연스러운 주목 유도 금지** | 현란한 애니메이션 등으로 광고에 시선을 끌 수 없음                             | 광고 주변에 애니메이션·화살표·강조 효과 일절 사용하지 않음                              |
| **광고 라벨링**                 | "Advertisements" 또는 "Sponsored Links"로만 표기 가능                         | AdSense 기본 라벨 유지, 커스텀 라벨 사용하지 않음                                       |
| **콘텐츠 위장 금지**            | 인접 콘텐츠를 광고처럼 보이게 포매팅할 수 없음                                | 광고 슬롯은 명확한 시각적 경계(배경색 구분 또는 구분선)로 콘텐츠와 분리                 |

### 5.2 광고 배치 설계

#### 5.2.1 멀티페이지 아키텍처 (핵심 전략)

**AdSense 정책에서 "유저가 요청한 새로고침"은 정상적인 광고 노출로 인정된다.** 따라서 본 서비스는 각 라운드를 별도의 페이지(또는 클라이언트 사이드 라우트 전환)로 구성하여, 유저가 "다음 라운드" 버튼을 클릭할 때마다 자연스러운 페이지 네비게이션이 발생하도록 설계한다. 이때 새 페이지에 포함된 광고 슬롯은 새로운 광고를 정상적으로 로드하며, 이는 AdSense 정책상 유저가 요청한 페이지 전환에 해당하여 정책 위반이 아니다.

```
[라운드 1 페이지]        [라운드 2 페이지]        [라운드 3 페이지]
/play?round=1      →    /play?round=2      →    /play?round=3
┌─────────────┐         ┌─────────────┐         ┌─────────────┐
│ 프로그레스 바 │         │ 프로그레스 바 │         │ 프로그레스 바 │
│             │         │             │         │             │
│ 썸네일 A    │         │ 썸네일 A    │         │ 썸네일 A    │
│             │         │             │         │             │
│    VS       │         │    VS       │         │    VS       │
│             │         │             │         │             │
│ 썸네일 B    │         │ 썸네일 B    │         │ 썸네일 B    │
│             │         │             │         │             │
│─────────────│         │─────────────│         │─────────────│
│ 광고 슬롯    │         │ 광고 슬롯    │         │ 광고 슬롯    │
│ (하단 배치)  │         │ (하단 배치)  │         │ (하단 배치)  │
└─────────────┘         └─────────────┘         └─────────────┘
     │                       │                       │
     └── 유저 클릭 "다음" ──→ └── 유저 클릭 "다음" ──→ └── ...
         (페이지 전환)            (페이지 전환)
```

**구현 방식**: Next.js App Router를 활용한 클라이언트 사이드 라우팅. URL이 `/play?round=N`으로 변경되며, 각 라운드 컴포넌트가 마운트될 때 AdSense 광고 컴포넌트가 자연스럽게 새 광고를 요청한다.

| 화면            | 슬롯 위치             | 광고 유형               | 비고                                  |
| --------------- | --------------------- | ----------------------- | ------------------------------------- |
| 랜딩 페이지     | 콘텐츠 하단           | 반응형 디스플레이 (1개) | CTA 버튼과 충분한 간격 확보           |
| 게임 라운드     | 콘텐츠 하단           | 반응형 디스플레이 (1개) | 썸네일과 150px+ 간격                  |
| 피드백 오버레이 | 없음                  | -                       | 피드백 화면에는 광고 미배치 (UX 우선) |
| 결과 화면       | 결과 카드 하단        | 반응형 디스플레이 (1개) | 공유 버튼과 간격 확보                 |
| 사이트 전역     | 앵커 광고 (하단 고정) | AdSense Anchor Ad       | Auto Ads 활용, 유저가 닫기 가능       |

### 5.3 광고 노출 시나리오 (정책 준수)

```
유저 1회 플레이 기준 광고 노출 흐름:

1. 랜딩 페이지 진입              → 하단 광고 1회 노출 + 앵커 광고
2. "테스트 시작" 클릭 (유저 액션) → 라운드 1 페이지 로드, 하단 광고 1회 노출
3. 라운드 1 답안 선택             → 피드백 표시 (광고 변경 없음)
4. "다음" 클릭 (유저 액션)        → 라운드 2 페이지 로드, 하단 광고 1회 노출
5. ... (라운드 3~20 반복)
6. 라운드 20 완료                → 결과 페이지 로드, 하단 광고 1회 노출

총 광고 노출 = 랜딩(1) + 라운드 페이지(20) + 결과(1) = 22회
(모두 유저의 명시적 클릭에 의한 페이지 전환으로 발생 → 정책 준수)

+ 앵커 광고는 전 페이지에 걸쳐 지속 노출 (AdSense Auto Ads 기본 기능, 정책 준수)
```

> **AdSense 정책 근거**: 유저가 직접 네비게이션한 페이지에서 새로운 광고가 로드되는 것은 "유저가 요청한 새로고침"에 해당한다. 이는 슬라이드쇼에서 유저가 다음 슬라이드로 넘길 때 새 광고가 로드되는 것과 동일한 원리로, AdSense가 명시적으로 허용하는 패턴이다.

### 6.2 AdSense 구현 가이드

#### 6.2.1 라운드별 광고 로드 (Next.js)

```typescript
// components/AdSlot.tsx
// 라운드 전환 시 컴포넌트가 리마운트되면서 자연스럽게 새 광고 요청
'use client';

import { useEffect, useRef } from 'react';
import { useSearchParams } from 'next/navigation';

export function AdSlot({ slotId }: { slotId: string }) {
  const adRef = useRef<HTMLDivElement>(null);
  const searchParams = useSearchParams();
  const round = searchParams.get('round');

  useEffect(() => {
    // 라운드(페이지) 전환 시 AdSense가 자연스럽게 새 광고를 로드
    // 자동 리프레시가 아닌, 컴포넌트 리마운트에 의한 새 요청
    if (typeof window !== 'undefined' && window.adsbygoogle) {
      try {
        (window.adsbygoogle = window.adsbygoogle || []).push({});
      } catch (e) {
        console.error('AdSense load error:', e);
      }
    }
  }, [round]); // round 변경 시 = 유저가 "다음" 버튼을 클릭한 것

  return (
    <div className="ad-container mt-[150px]"> {/* 콘텐츠와 150px+ 간격 */}
      <ins
        className="adsbygoogle"
        style={{ display: 'block' }}
        data-ad-client="ca-pub-XXXXXXX"
        data-ad-slot={slotId}
        data-ad-format="auto"
        data-full-width-responsive="true"
        ref={adRef}
      />
    </div>
  );
}
```

- 라운드별 이탈 데이터 기반 난이도 밸런싱

- 카테고리별 모드 (먹방 전문가, 게임 전문가 등)
- **Google Ad Manager + AdX 전환** (타이머 기반 리프레시 활성화, CPM 상향)

- 프리미엄 광고주 직접 영업

## 부록 A. AdSense 정책 준수 요약 (Quick Reference)

### ✅ 허용되는 것

- 유저가 클릭하여 다음 페이지로 이동할 때 새 페이지에서 광고가 로드되는 것
- 앵커 광고(하단 고정, 유저가 닫기 가능)
- 콘텐츠 하단에 적절한 간격을 두고 배치된 디스플레이 광고
- 반응형 광고 유닛
- Auto Ads (Google이 자동으로 최적 위치에 배치)

### ❌ 금지되는 것

- 타이머/setInterval 기반 광고 자동 새로고침
- 클릭 가능 요소(썸네일, 버튼) 사이에 광고 끼워넣기
- 현란한 애니메이션으로 광고에 시선 유도
- 광고 클릭을 유도하는 텍스트나 디자인
- 콘텐츠를 광고처럼 보이게 포매팅
- 팝업/팝언더 내 광고 배치
- 콘텐츠보다 광고가 많은 페이지
